# Pipeline: before-edit

Layers 1–6. Before writing code.

- Analyze the requirement and expected behavior.
- Discover existing code; reuse or extend before creating (`reviewers/architecture.md`).
- Validate fit with existing architecture, design system, and structure.
- Plan tests and locate reusable ones (`reviewers/tdd.md`).
- Write the smallest failing test first (Red).
- Judge effort; recommend a cheaper model if warranted (`policy/model-policy.md`).
- Check root + nearest local `CLAUDE.md`. Plan the smallest safe change.
