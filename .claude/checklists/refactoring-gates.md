# Gate: refactoring

- [ ] Refactoring was explicitly requested (not unprompted).
- [ ] Goal, scope, and behavior-to-preserve are clear — or were clarified first.
- [ ] Behavior unchanged: same tests pass before and after.
- [ ] No feature, fix, or unrelated change mixed in.
- [ ] Duplication removed; logic simplified; nesting reduced.
- [ ] Names clear and consistent with existing patterns.
- [ ] Stale comments updated or removed; survivors explain *why*.
- [ ] No dead code or stray files left behind.
- [ ] Any performance change is measured and its behavior risk justified.
- [ ] Change kept to small, reviewable steps.
